# Spike 2 E2E 验收

## 自动化 E2E

目标：

- 验证 App 能启动到实时记录工作台。
- 验证右侧翻译面板存在。
- 验证翻译面板初始状态为等待稳定转写。

命令：

```bash
xcodegen generate
xcodebuild test -project LiveNote.xcodeproj -scheme LiveNoteE2E -configuration Debug -destination 'platform=macOS'
```

当前状态：

- UI smoke test 已覆盖翻译面板初始状态。
- 当前本机命令行运行通过。
- 自动化不直接触发麦克风、语音识别资源下载和系统翻译资源下载，这些依赖系统权限与本机资源状态，需要人工验收。

## 人工 E2E

前置条件：

- macOS 26。
- Xcode 26。
- 系统麦克风可用。
- 中文语音识别资源可用，或允许系统下载。
- 中文到英文翻译资源可用，或允许系统准备翻译资源。

步骤：

1. 执行：

   ```bash
   xcodegen generate
   open LiveNote.xcodeproj
   ```

2. 在 Xcode 中选择 `LiveNote` scheme。
3. 运行 App。
4. 点击 `开始`。
5. 如出现麦克风权限弹窗，选择允许。
6. 对麦克风说一段中文，例如“大家好，我们今天讨论实时语音笔记。”
7. 等待稳定转写段落出现。
8. 验证该段落下方先显示 `翻译中`。
9. 等待翻译完成。
10. 验证该段落下方显示英文译文，并且右侧 `翻译` 面板同步显示原文与译文。
11. 点击 `停止`。
12. 验证已确认段落和英文译文仍然保留。

通过标准：

- App 不崩溃。
- 临时识别文本不会触发翻译。
- 稳定转写段落会触发翻译。
- 翻译成功后，中间段落和右侧翻译面板都显示译文。
- 翻译失败时，段落显示可理解的失败状态。

未通过时需要记录：

- 系统版本。
- 语音识别资源状态。
- 翻译资源状态。
- 麦克风权限状态。
- 错误提示。
- 是否能复现。
