# 第 2 周转写体验 E2E 验收

## 自动化 E2E

目标：

- 验证 App 能启动到实时记录工作台。
- 验证顶部状态胶囊、录音按钮和翻译面板初始状态存在。
- 验证会话侧边栏存在，并包含新建记录入口。
- 验证当前会话和最近会话在 UI 自动化中可访问。

命令：

```bash
xcodegen generate
xcodebuild test -project Kara.xcodeproj -scheme KaraE2E -configuration Debug -destination 'platform=macOS'
```

当前状态：

- `KaraE2E` 已覆盖启动、工作台、翻译面板、会话侧边栏、新建记录入口、当前会话和最近会话。
- 会话行已改为可点击列表行，便于后续接入会话切换，也让 macOS 辅助功能树稳定暴露行内容。
- 当前本机命令行运行通过。

## 单元测试验收

命令：

```bash
xcodebuild test -project Kara.xcodeproj -scheme Kara -configuration Debug -destination 'platform=macOS'
```

覆盖范围：

- 最终转写段落触发翻译。
- 临时转写段落不触发翻译。
- 编辑最终段落后标记为已编辑，并重新触发翻译。
- 标记重点后更新当前会话统计。
- 新建记录会归档已有会话并清空当前转写。

## 人工 E2E

前置条件：

- macOS 26。
- Xcode 26。
- 麦克风权限已允许，或测试时允许弹窗。
- 中文语音识别资源和中文到英文翻译资源可用，或允许系统准备资源。

步骤：

1. 执行：

   ```bash
   xcodegen generate
   open Kara.xcodeproj
   ```

2. 在 Xcode 中选择 `Kara` scheme 并运行 App。
3. 确认左侧显示当前会话、最近会话和 `新建记录`。
4. 点击 `开始` 并说一段中文。
5. 等待稳定转写段落出现。
6. 点击段落的星标按钮，验证该段落高亮，并且当前会话重点数量更新。
7. 点击段落编辑按钮，修改文本并保存。
8. 验证段落显示 `已编辑`，译文进入重新翻译状态，完成后显示新译文。
9. 点击 `停止`，验证当前会话进入完成状态并被归档到最近会话。
10. 点击 `新建记录`，验证当前工作台回到空状态。

通过标准：

- 会话列表、新建记录、当前会话和最近会话均可见。
- 稳定段落可编辑、可标记重点。
- 编辑后不会丢失用户修改，并会重新翻译。
- 新建记录不会覆盖已归档会话。
- App 不崩溃，失败状态有可理解提示。
