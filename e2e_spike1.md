# Spike 1 E2E 验收

## 自动化 E2E

目标：

- 验证 App 能启动到实时记录工作台。
- 验证初始状态显示为就绪。
- 验证开始按钮存在。
- 验证语言方向显示为中文到英文。

命令：

```bash
xcodegen generate
xcodebuild test -project LiveNote.xcodeproj -scheme LiveNoteE2E -configuration Debug -destination 'platform=macOS'
```

当前状态：

- UI 测试 target 已创建。
- UI smoke test 已添加。
- 当前本机命令行运行通过。
- 已验证 App 能启动到实时记录工作台，并显示初始状态、开始按钮与语言方向。

处理方式：

- 默认开发验证使用 `LiveNote` scheme 跑单元测试。
- E2E 使用独立 `LiveNoteE2E` scheme。
- 如果后续命令行 UI 自动化受系统权限限制，可以在 Xcode 中运行 `LiveNoteE2E`，或按下方人工 E2E 步骤验收。
- 菜单栏胶囊与屏幕截图类视觉验收仍需人工确认；当前自动化环境无法通过 `screencapture` 获取屏幕图像。

## 人工 E2E

前置条件：

- macOS 26。
- Xcode 26。
- 系统麦克风可用。
- 语音识别中文资源可用，或允许系统下载。

步骤：

1. 执行：

   ```bash
   xcodegen generate
   open LiveNote.xcodeproj
   ```

2. 在 Xcode 中选择 `LiveNote` scheme。
3. 运行 App。
4. 确认主界面出现：
   - 左侧 `LiveNote` 会话列表。
   - 中央 `准备开始实时记录`。
   - 顶部胶囊状态为 `就绪`。
   - 胶囊内显示 `中文 → 英文` 与 `本机`。
5. 确认 macOS 右上角菜单栏出现 LiveNote 横向胶囊状态项，而不是普通单个图标。
6. 胶囊状态项应包含：
   - 状态图标。
   - 录音时长，例如 `0s`、`1m 30s`。
   - 右侧状态 badge，例如 `Ready`、`REC`。
7. 点击菜单栏 LiveNote 胶囊，确认展开的是浮层状态卡片，而不是普通菜单；卡片内显示：
   - 当前状态。
   - 录音时长。
   - `中文 → 英文 · 本机`。
   - 开始 / 暂停与停止操作。
8. 点击 `开始`。
9. 如出现麦克风权限弹窗，选择允许。
10. 对麦克风说一段中文。
11. 验证中央区域出现 `识别中` 临时文本。
12. 停顿后验证文本转为 `已确认` 段落。
13. 点击 `停止`。
14. 验证窗口胶囊状态和菜单栏胶囊状态都回到就绪，并保留已确认段落。

通过标准：

- App 不崩溃。
- macOS 右上角菜单栏出现 LiveNote 横向胶囊状态项。
- 点击胶囊后展开状态卡片。
- 麦克风权限流程可触发。
- 录音状态能进入录音中。
- 中文语音能产生转写结果。
- 临时文本与已确认文本能区分显示。
- 停止后音频采集结束。

未通过时需要记录：

- 系统版本。
- 语音资源状态。
- 麦克风权限状态。
- 错误提示。
- 是否能复现。
