## Design Context

### Users
Kara is for macOS users who work with local coding agents such as Codex CLI, Claude CLI, and Hermes CLI. They use it from the menu bar while actively working in another app, usually to send a voice request with the current screen as context.

### Brand Personality
Direct, calm, and utility-first. The interface should feel like a native macOS control surface that helps the user recover quickly when a request fails.

### Aesthetic Direction
Use a restrained native macOS popover style with clear hierarchy, compact sections, and only the actions needed for the current request. Avoid dashboard-like navigation, decorative glass effects, marketing layouts, and exposing every configuration option on the first screen.

### Design Principles
- The first screen is for the latest voice request, not global configuration.
- Always show the retained transcript and screenshot preview when available.
- Show one primary recovery action first; keep secondary actions available but visually quieter.
- Permissions should be visible enough to diagnose capture failures without dominating the flow.
- Advanced setup belongs behind settings, not in the runtime panel.
